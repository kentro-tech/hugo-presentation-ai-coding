Iterative Development with AI

Build Better Code Through Small Steps & Active Learning

The Problem

AI Generates Code You Don't Understand

  • Large chunks of code appear instantly
  • You don't fully understand what was created
  • Technical debt accumulates quickly
  • You can't maintain or modify the code effectively
Why This Happens

AI Agents Make Bad Patterns Easy

  • ✖️ Encourage working in larger chunks
  • ✖️ Create code that seems to work but is bad
  • ✖️ Make being a passive participant easy
  • ✖️ Be very fun!

The path of least resistance leads to technical debt

The Solution

Spec-First Iterative Development

  • Define specifications before writing code (often just 2-4 steps)
  • Break work into small, testable steps
  • Build → Test → Review → Refactor after every step
  • Stay actively engaged and learning throughout

Step 1

Create Context

Provide AI with the tools and knowledge to build and verify

# Documentation
- LLMs.txt files for framework/library docs
- MCP servers for personalized context
- API documentation and examples

# Testing Tools
- Playwright for web development testing
- Profilers for performance validation
- Framework-specific test tools

AI needs both how to build and how to test that it works

Step 2

Create a Plan

Define the "what" — the high-level requirements

# Product Requirements

## Core User Experience
Three-column layout:
1. Cards (Left): Visual cards representing elements
2. Cycles (Center): Cards representing story beats
3. Outline (Right): Real-time visualization

## Data Model
- Cards table with schema: id, type, content, level
- Operations: add, edit, delete cards

Can be comprehensive or just the next 2-4 steps (a few minutes to write)

Step 3

Break Into Small Todos

Convert plan into specific, testable steps

# Development Plan

## Step 1: Verify starter app runs
- Ensure app runs successfully
- Verify you can access homepage
- Confirm correct environment

## Step 2: Create user input form
- Add form with text inputs
- Display form on homepage
- Verify form renders correctly

## Step 3: Save form inputs to database
- Create SQLite database and table
- Add endpoint to save form data
- Verify data persists after submission

Each step is small and verifiable (AI can help - often less than a minute)

Why Small Steps?

One Feature at a Time

  • Each todo adds one complete, testable feature
  • Small enough to understand fully
  • Easy to verify it works correctly
  • Easy to fix if something breaks

After each step, you test before proceeding

Step 4

The Build-Test-Verify Loop

1. Instruct: Ask AI to implement one todo
2. Test: Try it as a user would
3. Verify: Check it works correctly
4. Review: Read and understand the code
5. Learn: Research anything unfamiliar or curious, write that code yourself
6. Refactor: Improve before moving on

Repeat for every single todo

Critical Practice

Test After Every Single Todo

  • Don't batch multiple todos before testing
  • Test as a user would use it
  • Automated tests are good, but manual testing is essential
  • If it fails, fix it before moving forward

This catches problems immediately while context is fresh

After Testing

Manual Code Review

  • Read every line the AI generated
  • Understand what it does and why
  • Check against your style guidelines
  • Look for potential issues or improvements

Don't just accept code that "works"

Before Moving On

Refactor Manually

  • Improve code structure and clarity
  • Fix any issues you spotted in review
  • Ensure it matches your standards
  • Make it maintainable for the future

Only then move to the next todo

The Key Difference

Active Learning Throughout

  • You're engaged at every step
  • You understand what's being built
  • You learn from the code AI generates
  • You can maintain and modify it later

You're directing, not just watching

Benefits

Why This Process Works

  • Fast: Rapid prototyping with clear direction
  • Better Specs: Testing reveals what works
  • No Technical Debt: You understand everything built
  • Accessible: Non-coders can learn this approach
Summary

The Process

  1. Create context documentation
  2. Define a clear plan
  3. Break into small, testable todos
  4. For each todo:
    • Build it
    • Test it manually
    • Review the code
    • Refactor it
    • Then move to next

Stay actively engaged. Learn continuously.
